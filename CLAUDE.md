# Horsepower 🐎 (AI-Native Workflow Studio)

Real-time workshop app for "redesign, don't retrofit" exercises. Teams document a current business workflow (**Surface**), are surprise-swapped another team's torn-down workflow, and rebuild it AI-native (**Rebuild**) — with an LLM **Coach** challenging them throughout and a human facilitator (**the Farrier**) running the room.

**Two layers in this repo:** the running code is **v0.1** (the original model — see Architecture below); the **locked v0.2 design lives in `docs/specs/2026-06-11-*.md`** (`PRD-master` is the umbrella; `ux-end-to-end` is the build-from source of truth). Where they disagree, **the specs win** — v0.1 is what the rules below superseded.

## Product rules (do not break these — v0.2 locked methodology)

These encode the workshop methodology and are the point of the product:

1. **Capture ontology + WHY**: a workflow = trigger → inputs → personas → phases (each with "moments that matter", flaggable as pain points) → intent → outcome — **plus the WHY behind every element** (constraint raw-material). Intent must not be an artifact ("a report") — push for the decision it drives.
2. **The swap is a rotation AND a surprise** 🤫: at Rebuild, each team receives the *next* team's workflow — never foreshadowed. **Pre-reveal vocabulary rule** (capture spec §5a): no team-facing surface — UI copy, gate names, Coach utterances — says *swap / redesign / rebuild / hand over / receiving team / stranger / transfer* before the reveal. The team-facing gate is the **"Newcomer check"**; honest names live on the Farrier's private console. **The console is never projected** — only the **room view** (code + roster + timer).
3. **The teardown, not a handoff**: the receiving team gets an **abstract brief** (need/want) + **areas of concern** (problems, not steps) + **candidate constraints** + the **people inventory** (roster + capacities, never step-attached — the one scoped exception to zero-leak). All HOW is stripped, including the original layout (locked blocks arrive scrambled, Coach-placed). Teardowns are **pre-computed at gate-green** so the reveal is instant.
4. **Locked constraints**: intent, outcome, and **accountable/served** personas (not operators) are LOCKED in Rebuild — server-enforced; the *only* mutation path is a **Farrier-approved amendment** (logged on the brief). Never trust client payloads for locked fields.
5. **Anti-retrofit without leak-by-flag**: convergence with the hidden original is challenged generically to the team (process-cliché framing); the honest match-flag goes to the **Farrier console only**. Team-facing copy never references the hidden original.
6. **Human-landing gate**: every person in the people inventory must land — stays / transforms / removed-justified-by-the-design; "freed up for higher-value work" is rejected. Blocking = ✓-complete status + export marked partial — **the Farrier's timer always rules the room.**
7. **Phase state machine**: `lobby → surface → (swap) → rebuild → share → closed`, Farrier-driven, **phase-gated server-side** (Surface edits only in `surface`, Rebuild edits only in `rebuild`; swap needs ≥2 teams).
8. **Graceful degradation — including the gates**: any AI failure degrades to rule-based checks + question bank + full hand-authoring (the canvas is a real diagramming tool without the Coach). Gates are **system-owned and rule-evaluable offline**; a gate must never block a room because the AI is down. The workshop never stalls.
9. **B-lite collaboration**: each member on their own device; **one canonical AI-readable map per team** (server = source of truth); contributions reconciled through the Coach under the **contention rules** (human touch wins · ~10s cooldown · narrated settles · orphan-not-delete); direct edits node-scoped soft-lock/LWW + presence. No CRDT.

## Architecture (current code = **v0.2**, built 2026-06-12 per the specs)

- `server.js` (~480 lines): Express + `ws`. **Workshop {code, hostKey, state, teams[], timerEnd}** persisted to disk (`data/workshops.json`). State machine `lobby → surface → (swap) → rebuild → share → closed`, **phase-gated server-side**. Locked constraints (intent/outcome/accountable personas) server-enforced — the only mutation path is a Farrier-approved amendment. **Governance, the readiness gate, and the teardown are rule-based on the server → fully offline-capable** (the gate never blocks because the AI is down). Presence per member. AI proxy at `POST /api/coach` is enrichment-only and degrades to the question bank (200 + `degraded:true`).
- `public/index.html`: single-file vanilla-JS client, no build step. Design system inline (washes, paper grain, the §18 glyph defs + stamp copied from `craft-assets.html`, Fraunces/Inter/Caveat). **Layout (per `docs/specs/2026-06-12-coach-layout-and-motion-design.md`): the map is the full-bleed hero in both phases; the Coach is one collapsible/resizable right rail** (open in Surface, collapsed-with-unread-badge in Rebuild). Views: landing (host/join) · team picker · lobby (with cantering avatars + "meet the Coach" teaching vignettes) · Surface (**real diagramming canvas**: typed-block palette, two-click arrows, drag/resize/inline-edit, orphan tray, Newcomer-check gate) · swap-reveal stamp · Rebuild (**teardown lives ON the map as scattered context cards** — locked / candidate / concern, hover→WHY; people landed via a canvas "to land" tray; "Challenge this" on locked cards; always-visible Assumptions strip; "People N/M landed" pill) · Share (double reveal + diff + ledger + export) · Farrier console (**guided run bar: a display-only phase stepper + ONE primary "next step" CTA** that advances lobby→surface→rebuild→share→closed and greys out with the blocking reason, e.g. "need 2+ teams"; a guarded "↩ step back"; a **per-phase timer** (server-held `{durationMs,remainingMs,endsAt,running}`; pre-loads run-of-show defaults 20/30/10 min; load → ▶ Start → ⏸ Pause → ↺ Reset, broadcast to all + projector); a dedicated **lobby "set up your room" screen**; then the monitoring dashboard + live mirror + brief preview + amendments + present picker) · room view (projectable). The 🐎 **logo is a home control on every screen** (`goHome()`, confirm when a workshop is live). Phase-jumping is gone — forward-by-default; the swap is idempotent server-side so a step-back never re-seeds/clobbers Rebuild work. `makeCanvas()` (incl. `opts.overlays` for the teardown cards) is shared by Surface, Rebuild, the console mirror, and the share minis. Motion is GPU-only + `prefers-reduced-motion`-safe; the working canvas stays calm, play lives in waiting/transition moments.
- **Participant "game" layer** (per `docs/specs/2026-06-12-participant-experience.md`): a **randomised steed** (horse-name + colour, 🎲 re-roll, persisted, passed to the server so the Farrier roster/room view show it too) is your identity throughout; the team picker runs a **"meet the map" ontology tour**; the lobby is a **big "Meet the Coach" slide + your stable** (steeds canter in, no count fraction); participants get an **always-on timer + "final furlong" nudge**; **rosette** micro-win on gate-green; share-out **flips the constraint ledger** (🔒 kept / ✂️ MYTH) and hands out a **keepsake "race card."** Collaborative-only — no points/leaderboards (PRD). The stable metaphor is flavour-only; navigational names (Surface/Rebuild/Coach/Farrier) stay literal. Identity in localStorage; `editingLock` defers re-renders while a canvas label is being typed.
- AI provider: Anthropic (`ANTHROPIC_API_KEY`, default model `claude-opus-4-8`) or Azure OpenAI; `AI_PROVIDER` overrides. No key → built-in question bank; all rule-based checks + the whole canvas work offline.
- **Public-internet env (default-off; set on a public host — see `docs/DEPLOY.md`):** `ALLOWED_ORIGINS` (comma-sep WS origin allowlist; empty = allow all), `TRUSTED_PROXY_HOPS` (0 = direct/use socket addr, 1 = behind one PaaS proxy → read the proxy-appended XFF entry), `COACH_GLOBAL_MAX`/`COACH_IP_MAX` (AI key-spend caps), `MINT_GLOBAL_MAX` (room-creation backstop), `ANTHROPIC_BASE_URL` (endpoint override for testing/proxy). **Workshop codes are now 6 chars** (legacy 4-char rooms still join). Verified by `qa-online.js`.
- Reference mocks/prototypes in repo root (`mock-*.html`, `hifi-*.html`, `vibe-board*.html`, `craft-assets.html`, `journey-map.html`) — historical; the running v0.2 supersedes them.
